# Archived Realtime Refactor Plans

This directory stores superseded or completed plan documents from
`docs/realtime_refactor_plans/`.

Archive a plan here when:

- a newer plan revision replaces it
- the planned slice is complete and the detailed plan is no longer active
- a plan turned out to be wrong, incomplete, or based on outdated repo state

Keep archived plans rather than deleting them.
They provide a useful history of how the refactor evolved and help future
sessions understand why the current architecture took its present shape.

When archiving a plan:

- preserve the original filename when possible
- add a short note at the top of the archived file if it was superseded for a
  specific reason
- make sure the active checklist reflects the current status so the archive does
  not become the source of truth
